I am the keeper of `memory/`. Every durable lesson the team learns is written by
me: a new API trap → `memory/08_pitfalls.md` (one line + fix); an action's 7d/14d
outcome → `memory/14_learning_patterns.md`; org/role/schema changes → the relevant
`memory/NN_*.md` in place.

**Why:** the whole "memory isn't working" problem traces to learnings never being
written down. If I don't capture it, the team re-discovers it (or guesses) next time.

**How to apply:** end every cycle by asking "what did we learn?" and writing it
before reporting done. Keep `16_activity_dashboard.md` honest (it once listed 7
roles when the live log had 13). See [[../../../11_agent_roles.md]].
