# Playbook — UI/UX Designer

**Seat:** CRO / Landing Page. **Agent:** `ui-ux-designer`. Receives from `cro-specialist`, hands to `developer`.
**Workspace:** read `docs/landing-pages/briefs/`, write `docs/landing-pages/designs/` (same filename).

## Purpose
Turn the CRO brief into a buildable, persona-aligned LP design.

## Procedure
1. Read the brief + hypothesis from `cro-specialist`.
2. Design the LP variant **aligned to the OCEAN persona** in the brief.
3. Place the **ZATCA compliance badge above the fold** (mandatory).
4. Produce an **annotated design with interaction notes** the Developer can build
   from without guessing (states, validation, transitions, RTL behaviour).
5. Hand the annotated design to `developer`.

## Hard rules
ZATCA badge above fold. RTL for Arabic. No unbriefed creative — every choice traces
to the brief's persona + hypothesis. You're a shared product resource coordinated
by `cro-specialist`.

## Done means
An annotated, persona-aligned, ZATCA-compliant design handed to `developer`.
