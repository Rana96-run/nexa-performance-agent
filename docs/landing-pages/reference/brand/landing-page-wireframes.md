## Build conventions

When the `pages-ar` / `pages-en` agents build pages from this spec:

1. **Hero trust signal** — number is configurable (`25,000+` for SMB, `1,000+` for accounting teams). Pull from current data; never hardcode an outdated number.
2. **Pricing prices** — placeholders (`X /mo`, `125 /mo`, `175 /mo`) are NOT final. Source the live numbers from `https://www.qoyod.com/pricing/` before publishing. If unknown, leave as `—` and surface a TODO in the chat reply rather than inventing a price.
3. **Testimonials** — only use names and quotes from the approved list in `value-proposition.md` § The Proof. Never invent a testimonial.
4. **Cross-product cards** ("Solutions to Boost Your Growth") — link the two cards (POS + Services) to the actual product pages once they exist. If they don't yet exist, link to the parent product/services page.
5. **Topical alignment isolation test** (per `topical-alignment.md`) — every section eyebrow + H2 must pass the test. The wireframes here are pre-validated, but if you adapt them for sub-pages, re-test each snippet.
6. **Arabic builds** — translate via `arabic-writing-style.md`. Watch for translatese in the section headlines (e.g., "Take control of your finances" → "تحكّم في أموالك" not "خذ السيطرة على أموالك").
