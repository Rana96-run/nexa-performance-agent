## Section B — Why Python can't be replaced

Each item below is a concrete capability that cannot move to n8n, a spreadsheet formula, or a no-code tool.

1. **OAuth token refresh (LinkedIn, TikTok, YouTube)** — Multi-step handshake: read current refresh token from BQ `platform_tokens`, call the platform auth endpoint with client_id + client_secret + refresh_token, receive a new access token + new refresh token, write both back to BQ before making any API call. This is a stateful, multi-step operation with a persistent token store. n8n's HTTP Request node can make one call but cannot read/write BQ for the token and handle the refresh-before-expiry logic atomically. Failure mid-handshake must be caught and re-tried — not possible in n8n without complex error branches.

2. **BQ NDJSON upserts (MERGE)** — All collectors write via `bq_writer.py::upsert_rows()` which serializes rows to NDJSON, loads to a temp table, then runs a `MERGE` statement against the target. n8n has no BQ MERGE node. The BQ node in n8n only supports `INSERT` — which creates duplicates on re-run. Streaming inserts (the other n8n option) lock rows for 90 minutes and block `DELETE`, making idempotent re-runs impossible. Python + load jobs is the only safe upsert path.

3. **Google Ads API cursor-based pagination** — The Google Ads API returns 10,000 rows per page with a `next_page_token`. A full keyword history pull for one account can be 50–200 pages. Python's `google-ads` SDK handles the cursor automatically; each page must be fetched, accumulated, and bulk-loaded into BQ. n8n's loop nodes have no SDK and cannot accumulate 200 pages into a single load job.

4. **Keyword policy enforcement (`keyword_policy.py`)** — 200+ rules covering: ALWAYS_NEGATIVE patterns (bilingual Arabic + English: login/تسجيل الدخول, free/مجاني, course/دورة, etc.), BRAND_ONLY terms (قيود + accounting modifier disambiguation), COMPETITOR terms (Foodics/فودكس, Daftra/دفترة, etc. — must not be negated), language-mismatch detection (Arabic keyword in English campaign), 30-keyword cap per ad group, minimum 10-day age before pause eligibility, converting-keyword exception (QS<5 but conv>4 and $10≤CPA≤$70 → leave enabled). This is a decision tree with bilingual regex matching and BQ lookups. It cannot be expressed as n8n conditions.

5. **Ad pause/scale execution (atomic: SDK + approval gate + BQ audit log)** — A pause operation requires: (1) check `#approvals` Slack channel for ✅ reaction on the specific task, (2) call the platform SDK to pause the ad/campaign, (3) write an audit row to `agent_activity_log` in BQ. All three must succeed atomically — if the SDK call fails, the audit log must reflect it, and the approval state must not be consumed. n8n cannot implement this three-way atomic transaction with rollback.

6. **HubSpot CDC (change detection) across 18 months** — The HubSpot leads and deals collectors do a full-window re-pull from 2025-01-01 every 6h: paginate HubSpot's search API (cursor-based, 100 records/page, up to thousands of pages), detect changes by comparing `hs_lastmodifieddate` against the BQ mirror, and upsert by stable `hs_object_id`. The 18-month window means 20,000+ records per sync. HubSpot's search API has property-level field selection, association fetching, and rate limiting (10 req/s). Python handles all of this with retry logic. n8n's HubSpot node fetches single objects or simple lists — it cannot do a cursor-paginated full-window sync with upsert.

7. **Forecasting + period comparison** — `analysers/forecaster.py` and `analysers/period_compare.py` run multi-step calculations: pull two date windows from BQ, compute week-over-week deltas for 6+ KPIs, flag regressions vs improvements, project end-of-month spend/leads/CPQL based on current run rate and historical seasonality. These are ~200-line Python functions with conditional logic, statistical math (SAFE_DIVIDE, compound growth rates), and formatted Slack output. They cannot be expressed as SQL alone (would require repeated CTEs and lose the projection logic) and cannot be done in n8n without becoming unmaintainable YAML.
