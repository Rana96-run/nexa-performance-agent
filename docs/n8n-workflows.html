Cadence Workflows

3 workflows

cadence_daily

⏰ Daily 07:00 Riyadh
Purpose Full daily intelligence loop — pulls ad data from all 6 platforms, reconciles with HubSpot, runs 8-step analysis via 3 Claude agents, posts nightly digest.
Flow
  1. Calls infra_data_collection → pulls yesterday's metrics from all 6 platforms into BQ
  2. BQ ↔ HubSpot reconciliation (must be within 2% delta — aborts if not)
  3. Three Claude agents run the 8-step intelligence loop
  4. Routes KPI flags to 6 KPI sub-workflows based on threshold breaches
  5. Posts nightly digest to #notify and #approvals with Asana tasks per action
Reads
campaigns_daily hubspot_leads_module_daily ads_daily
Writes
Slack #notify Slack #approvals Asana tasks (per channel)
Calls
infra_data_collection infra_qa_gate kpi_cpql kpi_cpl kpi_roas kpi_qual_ratio kpi_impression_share kpi_creative_ctr

KPI Sub-flows

6 workflows

Called by cadence_daily when a KPI threshold is breached. Each sub-flow runs its own BQ query, passes results to a dedicated Claude analyst node, and returns a structured Asana task + recommendation JSON to the parent workflow.

kpi_cpql

Called by cadence_daily
Fires when Channel CPQL > $60 Action at CPQL > $95 per campaign
Logic
  • Queries 30-day BQ data using campaigns_daily + hubspot_leads_module_daily CTE join (pre-aggregated to prevent spend fan-out)
  • Passes worst campaigns to Claude CPQL Analyst node
  • Returns root-cause JSON + [CPQL FIX] Asana task with full campaign path, period comparison, and recommended action
Called by cadence_daily

kpi_cpl

Called by cadence_daily
Fires when Channel CPL > $25 Action at CPL > $38 per campaign
Logic
  • 30-day CTE query on campaigns_daily joined to hubspot_leads_module_daily
  • Passes results to Claude CPL Analyst node
  • Flags LP quality issues with a CRO-handoff marker if the issue traces to the landing page
  • Returns [CPL FIX] Asana task with root-cause attribution and full setup recommendation
Called by cadence_daily

kpi_roas

Called by cadence_daily
Fires when Channel ROAS < 1.0×
Logic
  • 3-factor check: qual ≥ 45%, CPQL ≤ $85, volume vs prior period
  • All three factors green → creates Sales Escalation task (ROAS issue is downstream of pipeline, not media)
  • Any factor red → creates Campaign Fix task for performance-lead with root-cause attribution
Called by cadence_daily

kpi_qual_ratio

Called by cadence_daily
Fires when Qualification rate < 45%
Severity Split
  • P0  Qual ≤ 30% → urgent LP-redirect task, due today, escalated to cro-specialist seat
  • P1  Qual 30–45% → qual-improvement task with worst campaigns listed + root-cause hypotheses for team review
Called by cadence_daily

kpi_impression_share

Called by cadence_daily
Fires when Impression share flag fires
Logic
  • Queries 14-day avg IS, lost_is_budget, and lost_is_rank from campaigns_daily (paused campaigns excluded)
  • Claude IS Analyst node determines whether root cause is budget-constrained vs rank/quality-constrained
  • Outputs per-campaign bid and budget recommendations with expected IS uplift
Called by cadence_daily

kpi_creative_ctr

Called by cadence_daily
Fires when Creative CTR decay flag fires
Logic
  • Queries ads_daily — 3-day CTR vs 7-day baseline; filters to >20% CTR drop with ≥ 1,000 impressions
  • Claude Creative Analyst node classifies each ad as TWEAK (adjust creative) or KILL (retire and replace)
  • Zero-conv + CTR decay combination = immediate pause recommendation + new creative brief in Asana
Called by cadence_daily

Infrastructure

4 workflows

infra_data_collection

Called by cadence_daily
Purpose Single data-ingestion gateway — hits all 6 ad platform APIs in parallel, normalises spend to USD, merges into campaigns_daily. Platforms
Google Ads (2 accounts) Meta Snapchat TikTok Microsoft LinkedIn
Key Rules
  • Spend normalised to USD — Google Ads cost_micros and Snap spend divided by 1,000,000
  • Each platform node has continue-on-error — one platform failure does not block the others
  • Microsoft connector handles OAuth token self-rotation without manual re-auth
Writes campaigns_daily (BQ MERGE) Called by cadence_daily

infra_approval_listener

Slack Events API — always-on
Purpose Human-in-the-loop gate for all pause/scale actions. Listens for emoji reactions on #approvals messages and resumes or cancels the pending n8n execution.
How it works
  • Handles Slack URL-verification challenge automatically on first connect
  • ✅ reaction on an #approvals message → resumes the suspended n8n execution via approval webhook
  • ❌ reaction → posts "Action rejected — skipped" in the message thread, then closes the execution cleanly
Always-on requirement: This workflow must be active at all times. Pause/scale approvals fail silently if this workflow is off — the parent execution waits indefinitely for a webhook that never fires.

infra_qa_gate

Called by all cadence workflows
Purpose System-wide compliance gate — validates that every output from a cadence workflow is structurally correct and policy-compliant before it reaches Slack or Asana.
Checks
  • Required fields present: sub_flow and channel
  • Valid status value (must be an allowed enum)
  • No SAR-denominated spend in output (USD only — enforcement of currency policy)
  • No auto-execution claims in output text (all actions must be approval-gated)
Returns QA_PASSED — forwards payload downstream  |  QA_FAILED — returns structured error list, halts execution Called by
cadence_daily cadence_weekly cadence_monthly

infra_data_health

⏰ Daily 06:00 UTC (09:00 Riyadh)
Purpose BQ ↔ HubSpot sync health monitor — compares BQ row counts against live HubSpot API for deals and leads. Posts ✅ or ❌ per metric to #data-health.
What it checks
  • hubspot_deals_daily + hubspot_leads_module_daily vs HubSpot API — last 60 days, paid channels, 3 pipelines
  • Coverage ratio threshold: < 85% for either deals or leads triggers ❌ alert
Writes Slack #data-health with ✅/❌ per metric
