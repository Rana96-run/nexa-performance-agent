🎭 Agent Roles · 5 active skills

Paid Media Analyst
10-year MENA SaaS analyst · CPQL-first
Runs the 8-step intelligence loop on every analysis: Observe → Compare → Investigate → Decide → Execute → Monitor → Learn → Forecast. Outputs two layers — a CEO summary and a full technical breakdown.
Load when
  • Any CPQL analysis or period comparison
  • Daily Slack digest creation
  • Campaign health check
  • "Why did X happen?" questions
🔍
Connector Police
Data integrity officer · 5-check sweep
Runs 5 checks per connector: freshness, row integrity, spend sanity, attribution health, credentials. Posts to #nexa-health only when BROKEN. Runs nightly at 08:30 Riyadh as part of the operational scheduler.
Load when
  • Data gaps or empty tables
  • Dashboard shows wrong numbers
  • Connector health errors in logs
  • New integration setup
⚙️
Data Engineer
BQ architect · load-job-only rule
Manages BigQuery schema, table creation, view rebuilds, and backfills. Strict rules: no streaming inserts, always load_table_from_file, minimum 7-day reconciliation before declaring done on any schema change.
Load when
  • BQ schema changes or new tables
  • View rebuilds after migrations
  • Backfill requests
  • Collector refactoring
💰
Media Buyer
Campaign architect · naming enforcer
Creates campaigns with the full 12-field spec: naming convention, audience architecture (prospecting/retargeting/exclusions), pixel setup, UTM passthrough, LP assignment. Never creates without approval.
Load when
  • New campaign creation or cloning
  • Audience setup or scaling
  • Naming convention validation
  • Campaign brief review
🎯
CRO Paid Specialist
LP conversion · CPQL-to-LP loop
Bridges campaign CPQL data from the Nexa agent to landing page performance from the LP Agent. Audits existing pages, builds new LP specs (8-section template), and runs A/B tests. CPQL is the only success metric.
Load when
  • New paid campaign needs an LP
  • CPQL regression traced to LP quality
  • A/B test definition
  • Sector LP audit
✅ How Approvals Work
1
Nightly Audit
Agent runs at 08:00 Riyadh. Finds scale + pause candidates across all channels.
2
Asana Tasks
One Asana task per finding — full setup, CPQL data, date range, forecast.
3
#approvals Digest
Single Slack message with all scale + pause items. Review-only items go to Asana only.
4
React ✅ or ❌
✅ executes ALL scale + pause items. ❌ skips all. Comment to exclude individual items.
5
Execute + Log
Pauses fire first, then scales. Every action logged to BQ and Asana task updated.
6
14-Day Check
Agent re-evaluates CPQL at 7d and 14d. Outcome written to memory for next session.
Rule: Scale and pause actions ALWAYS need ✅ in #approvals. The agent never auto-executes. Exception: negative keyword additions (zero spend at risk).
